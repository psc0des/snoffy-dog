# 🐕 Snoffy Dog - AI-Powered Security Scanner

A VS Code extension that sniffs out security vulnerabilities in your code with personality and intelligence!

Built for the **IBM BoB Hackathon** - combining traditional security scanning with IBM watsonx.ai Granite models for intelligent false-positive filtering.

## Problem Statement

Security scanners flood developers with false positives — placeholder API keys, test fixtures, and example values get flagged the same as real secrets. Most scanners are dry, ignored, or tuned out, leading to real leaks slipping through.

## Solution

Snoffy Dog combines a 22-rule regex scanner with **IBM watsonx.ai Granite-4-h-small** as an intelligent false-positive filter. Every regex match is sent to Granite with the file path, the matched snippet, and surrounding code; Granite returns a structured verdict (real / placeholder / uncertain) with reasoning. Findings are grouped by verdict in a delightful sidebar with a dog mascot — making security review fast, accurate, and actually pleasant.

## What Snoffy Looks For

Snoffy's rule pack covers **22 security patterns** across three categories:

### 🔑 Hardcoded Secrets
- AWS Access Key IDs (`AKIA...`)
- AWS Secret Access Keys
- Generic API tokens (variables named `api_key`, `token`, `secret`)
- JWT tokens (3 base64 parts joined by dots)
- Private key blocks (`-----BEGIN PRIVATE KEY-----`)
- Hardcoded passwords
- GitHub Personal Access Tokens (`ghp_...`)
- Slack tokens (`xox...`)
- Stripe live API keys (`sk_live_...`)

### ⚠️ Dangerous Code Patterns
- `eval()` usage
- `exec()` calls
- Raw SQL string concatenation
- SQL query string building
- Command injection risks
- Unsafe deserialization (`pickle.loads`, `yaml.load`)

### 🌐 Development & Staging URLs
- `localhost` references
- Dev URLs (`dev.domain.com`)
- Staging URLs (`staging.domain.com`)
- Test URLs (`test.domain.com`)
- Internal IP addresses (`192.168.x.x`, `10.x.x.x`)
- Example/placeholder URLs

For full rule documentation, see [SCANNER-RULES.md](./SCANNER-RULES.md).

## AI-Powered Intelligence

Snoffy Dog uses **IBM watsonx.ai Granite models** to intelligently analyze each finding:

### How It Works

1. **Pattern Detection**: Regex-based scanning finds potential security issues
2. **Context Extraction**: Captures suspicious code with surrounding context
3. **AI Classification**: Granite AI analyzes each finding in batches
4. **Confidence Scoring**: Returns whether findings are real threats or false positives
5. **Smart Display**: Organizes results into four categories:
   - ✓ **Verified Issues** (High confidence, real threats)
   - ? **Needs Review** (Uncertain, requires human judgment)
   - ✗ **Likely False Positives** (High confidence, safe to ignore)
   - ⚠ **Errors** (Granite call failed — finding shown but unverified)

### Example Classifications

```javascript
// ✓ VERIFIED - Real security issue
const apiKey = "sk-1234567890abcdef1234567890abcdef";

// ✗ FILTERED - Placeholder/example text
const exampleKey = "your-api-key-here";
const testToken = "xxx-yyy-zzz";

// ? NEEDS REVIEW - Uncertain context
const key = process.env.API_KEY || "default";
```

## Installation

There are three ways to install Snoffy Dog. Pick whichever matches your situation.

### Option A — Install the prebuilt VSIX (fastest)

The repo ships a prebuilt `snoffy-dog-0.1.0.vsix` you can install directly into VS Code or any VS Code-compatible IDE (including IBM Bob IDE).

1. Clone or download this repo
2. Open VS Code
3. Open the **Extensions** panel (`Ctrl+Shift+X`)
4. Click the **`...`** menu in the top-right of the Extensions panel
5. Choose **"Install from VSIX..."**
6. Select `snoffy-dog-0.1.0.vsix` from the repo root
7. Reload VS Code when prompted

### Option B — Build the VSIX from source

If you want to inspect or modify the code before installing:

```bash
# Clone the repo
git clone https://github.com/psc0des/snoffy-dog.git
cd snoffy-dog

# Install dependencies and compile
npm install
npm run compile

# Install vsce (one-time, global)
npm install -g @vscode/vsce

# Package the extension into a .vsix
vsce package
# → produces snoffy-dog-0.1.0.vsix
```

Then follow steps 2–7 of **Option A** to install the resulting `.vsix`.

### Option C — Run from source (for development)

To debug or modify the extension live:

```bash
git clone https://github.com/psc0des/snoffy-dog.git
cd snoffy-dog
npm install
npm run compile
```

Open the folder in VS Code and press **F5**. This launches an Extension Development Host window with Snoffy Dog loaded.

## Configuration

### Quick Start (Without AI)

The extension works out-of-the-box with regex-based scanning. All findings will be marked as "Needs Review" (since Granite filtering is disabled).

### Enable AI Filtering (Recommended)

To enable intelligent false-positive filtering with IBM watsonx.ai, set three environment variables **before launching VS Code** so the extension picks them up at startup.

1. Get your IBM Cloud credentials (see [WATSONX_SETUP.md](./WATSONX_SETUP.md))

2. Set environment variables:

   **Windows (PowerShell — current session):**
   ```powershell
   $env:WATSONX_API_KEY = "your-ibm-cloud-api-key"
   $env:WATSONX_PROJECT_ID = "your-watsonx-project-id"
   $env:WATSONX_URL = "us-south.ml.cloud.ibm.com"
   ```

   **Windows (System-wide, persistent):**
   - Press `Win+R` → type `sysdm.cpl` → Advanced → Environment Variables
   - Under **User variables**, add `WATSONX_API_KEY`, `WATSONX_PROJECT_ID`, `WATSONX_URL`
   - Restart VS Code

   **macOS / Linux (bash/zsh):**
   ```bash
   export WATSONX_API_KEY="your-ibm-cloud-api-key"
   export WATSONX_PROJECT_ID="your-watsonx-project-id"
   export WATSONX_URL="us-south.ml.cloud.ibm.com"
   ```
   Add these lines to `~/.bashrc` or `~/.zshrc` to persist across sessions.

3. Launch VS Code from the same shell (so it inherits the env vars), or restart VS Code if you set them system-wide

4. Open any workspace and click **"Go Sniffing!"** in the Snoffy panel

> **Note on `WATSONX_URL`:** Use the bare hostname (`us-south.ml.cloud.ibm.com`), NOT a full URL with `https://`. The extension adds the protocol itself; including it twice causes DNS hangs.

📖 **Full setup guide**: [WATSONX_SETUP.md](./WATSONX_SETUP.md)

## Usage

1. Open the Snoffy Dog panel from the Activity Bar (look for the dog icon 🐕)
2. Click "Go Sniffing!" to start a security scan
3. Watch Snoffy walk as he scans your files
4. Review findings organized by verification status:
   - Click verified issues to fix them immediately
   - Review uncertain findings with human judgment
   - Expand filtered findings if you want to double-check
5. Click on any finding to navigate to the code with animated paw prints!

## Built with IBM Bob

This extension was built end-to-end by **IBM Bob** in a single day using ~36 Bobcoins. Bob wrote the TypeScript, the webview, the animations, the watsonx.ai integration, the report panel — everything.

The build was iterative: ~10 major prompts plus several short debugging rounds. Real moments where Bob's repository-wide context paid off:

- Diagnosed why Granite returned empty responses by reading the API call code, then fixed the deprecated `/ml/v1/text/generation` endpoint to `/ml/v1/chat/completions` in one targeted change
- Caught that an `https://` prefix on `WATSONX_URL` was hanging DNS — fixed by stripping the protocol defensively
- Found and fixed a click handler bug where Granite reasons containing apostrophes broke the inline `onclick` — rewired with event delegation

Full task transcripts (every prompt, every Bob response) are exported in the **`bob_sessions/`** folder for the hackathon judges.
